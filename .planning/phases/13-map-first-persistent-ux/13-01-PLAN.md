# Plan 13-01: Persistent Map Component

## Task

Create a single PersistentMap component that persists across all session stages and handles marker rendering based on stage.

## Implementation

1. Create `src/components/map/PersistentMap.tsx`
2. Accept props:
   - `stage`: "setup" | "spots" | "shortlist" | "confirm"
   - `userLocation`: { lat, lng } (setup stage)
   - `results`: RankedVenue[] (spots stage)
   - `shortlistVenueIds`: string[]
   - `confirmedVenueId`: string | null
   - `selectedVenueId`: string | null
   - `onMarkerSelect`: callback
   - `onVenueFocus`: callback
3. Marker behavior by stage:
   - setup: Show user pin marker
   - spots: Show venue markers (colored by state), zoom to fit bounds
   - shortlist/confirm: Venue markers with shortlist/confirmed states

## Stage Transitions

- When stage changes to "spots", call fitBounds to zoom to cover all results
- User pin fades/removes when transitioning from setup
- Markers animate in with appropriate colors

## Key Features

- Uses existing @vis.gl/react-google-maps
- Leverages AutoFitBounds from RankedSpotsMap
- Marker styling from existing components
