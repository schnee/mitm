# GSD New Project Draft

## Purpose
Use this prompt with `/gsd-new-project` to generate an implementation-ready product roadmap for **Meet Me in the Middle**, including phased delivery, measurable outcomes, and clear scope boundaries.

## Copy/Paste Prompt
```text
/gsd-new-project

Create a practical roadmap for a new product called "Meet Me in the Middle."

Product context:
- Problem: Two people in different locations need a fair, fast way to pick a place to meet.
- Inputs: each person location, willingness-to-travel percentage split (for example 70/30), and high-level preference tags (coffee, cocktails, vintage shops, etc.).
- Output: ranked places/POIs between them.
- Primary use cases: networking and dating.
- Platforms: mobile and web.

What I need from you:
1) A phased roadmap from discovery through launch and stabilization.
2) For each phase, include:
   - objective
   - core deliverables
   - success metrics (leading and lagging)
   - major risks
   - assumptions
   - explicit non-goals
3) Define 30-day outcomes and 90-day outcomes with concrete target signals.
4) Call out decisions that must be made early (product, UX, data, and technical).
5) Include a short dependency map and sequencing rationale.
6) End with a concise "first 2 weeks" execution plan.

Constraints:
- Keep it concise and implementation-ready.
- Prioritize high-leverage scope for an MVP first, then expansion.
- Make tradeoffs explicit when suggesting sequence.
```

## Customization Knobs
- Replace target audience emphasis (networking-first, dating-first, or balanced).
- Set team size and roles to tune phase granularity.
- Add geography limits (single city, multi-city, global) for rollout assumptions.
- Add specific business goals (retention, conversion, paid plans) to shape metrics.
